## Brand & Style
The design system is engineered for an academic environment, prioritizing clarity and intellectual calm. It balances the precision of cryptographic algorithms with a premium, editorial aesthetic. The visual language moves away from the typical "hacker" trope, instead adopting a sophisticated, gallery-like presentation.

The style is **Minimalist-Modern** with **Tactile accents**. It uses generous whitespace and a restricted palette to focus the user’s attention on the mathematical beauty of the AES-128 process. Interaction is designed to feel deliberate and smooth, evoking the sensation of handling high-quality physical instruments.

## Colors
The palette is rooted in a warm, paper-like foundation (#FDFBF7) to reduce eye strain during long study sessions. 

- **Primary & Secondary:** Copper (#B87333) and Silver (#C0C0C0) are reserved for 3D state visualizations and metallic UI accents. Indigo (#3D5A80) serves as the primary interactive anchor for CTAs.
- **Functional Colors:** A specific quartet of colors is used to visually categorize the four core AES operations. These should be used consistently in state matrices, flow diagrams, and text labels to reinforce mental mapping.
- **Neutrality:** Use low-contrast greys for borders and secondary text to maintain the editorial feel.

## Typography
This design system utilizes a tri-font strategy to separate narrative, interface, and data layers.

- **The Narrative Layer:** Playfair Display is used for page titles and section headers, providing a literary and authoritative tone.
- **The Interface Layer:** Geist is used for all UI labels, descriptions, and navigation. Its geometric clarity ensures the platform feels modern and technical.
- **The Data Layer:** JetBrains Mono is strictly reserved for Hexadecimal values, bitstrings, and code snippets. The high x-height and distinct characters prevent ambiguity between '0' and 'O'.

## Layout & Spacing
The layout follows a **Fixed Grid** approach for the main content area to mimic the structured nature of cryptography. 

- **State Matrices:** The core 4x4 AES matrices should use a tight 4px gap to appear as a single cohesive unit, while larger sections utilize a 24px gutter.
- **Responsive Behavior:** On desktop, the simulator utilizes a multi-pane layout (Input / Transformation / Output). On mobile, these reflow into a vertical stack with "step-by-step" navigation.
- **Padding:** Apply generous vertical padding (64px+) between major sections to maintain the editorial "breathability."

## Elevation & Depth
Depth is used sparingly and functionally. 
- **Surface:** The primary background is flat.
- **Tonal Layers:** Subtle shifts in background color (e.g., from cream to a slightly cooler off-white) are used to distinguish the "Laboratory" area from the "Instructions" sidebar.
- **Shadows:** Use extremely soft, long shadows for the active state matrix to suggest it is "floating" above the desk. 
- **Active States:** Elements being transformed should have a subtle outer glow using the operation-specific color (e.g., an Amber glow during SubBytes).

## Shapes
The shape language is defined by **Pill-shaped** containers for interactive elements and **Soft-rectangles** for data containers.

- **Buttons & Chips:** Use a full border-radius (pill) for all primary buttons and status tags.
- **Matrices:** Individual hex cells in a matrix should have a small 4px radius (`rounded-sm`) to maintain a grid-like appearance while avoiding "sharp" corners.
- **Cards:** Main content cards use `rounded-xl` (1.5rem) to soften the technical nature of the content.

## Components
Consistent implementation of components is vital for educational clarity.

- **Buttons:** Primary buttons are pill-shaped, filled with Indigo (#3D5A80), and use white Geist medium text. Secondary buttons use a copper outline.
- **The Matrix:** A 4x4 grid of Geist/JetBrains Mono cells. Each cell has a background transition that triggers during operations.
- **Input Fields:** Minimalist underlines or very light-grey containers with no heavy borders. Focus states use a 2px copper bottom-border.
- **Progress Stepper:** A horizontal track at the top of the UI showing the 10/12/14 rounds of AES. Active rounds are highlighted with a metallic silver gradient.
- **Chips:** Small, pill-shaped tags used to indicate the current AES phase (e.g., "Round 1 - SubBytes") using the semantic operation colors.
- **Tooltips:** Rich tooltips with JetBrains Mono text to explain specific hex transformations on hover.
